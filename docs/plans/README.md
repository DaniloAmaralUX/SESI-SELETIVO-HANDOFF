# docs/plans

Planos de implementação — saídas de **`/ce-plan`** (e o alvo de **`/ce-work`** / **`/lfg`**).

Um plano descreve o **como**: unidades de implementação, arquivos afetados, padrões a reusar, verificação.
Nasce de um brainstorm em [`../brainstorms/`](../brainstorms/) ou de um requisito, e é enriquecido até ficar
*implementation-ready* antes de virar código.

> Convenção: `AAAA-MM-DD-NNN-<tipo>-<slug>-plan.md`, caminhos **relativos ao repo**. `/ce-work` só executa
> planos marcados implementation-ready + execução de código.
